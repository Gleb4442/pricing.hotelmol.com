1) Общая цель
Собрать статический одностраничный лендинг (HTML/CSS/JS) в Replit, визуально повторяющий референс по сетке/типографике/карточкам тарифов, с внесёнными правками цен, переключателем режима биллинга и подсказками по ховеру. Исходный макет и классы берём из текущего index.html.
2) Структура проекта
/ (корень Replit)
  index.html        // Разметка, вкл. шаблоны подсказок
  styles.css        // Стили, вкл. состояние usage/monthly
  script.js         // Логика тумблера, тултипов, кнопок оплаты
  /assets           // Изображения и иконки
  /fonts (опц.)     // Если шрифты локально; иначе Google Fonts
3) Контент и цены (строго)
PRO (оплата за использование)
Старая цена (зачёркнутая): «8 центов»
Текущая цена: «7 центов за запрос»
Фикс помесячно (при переключении режима): старая «459 долларов» (зачёркнутая), текущая «399 долларов в месяц».
PREMIUM
Оплата за использование: «30 центов за запрос»
Фикс помесячно: «1499 долларов в месяц».
Важно: писать стоимости словами («7 центов», «30 центов», «399 долларов»), а не цифрами с точками.
4) Переключатель биллинга (UI/UX)
В шапке — тумблер: «Оплата за использование» ↔ «Фиксированная помесячно».
Состояние хранится в localStorage (billingMode = 'usage' | 'monthly') и в URL-якоре #billing=usage|monthly.
При смене режима:
Обновляются тексты цен на карточках.
Появляется/скрывается боковой инфоблок (см. раздел 5).
У опций «персональный Telegram-бот» и «удаление логотипа» — бейджи «+0,5 цента/запрос» (usage) или «Входит» (monthly).
5) Боковой инфоблок для usage-режима
Десктоп: правый фиксированный (sticky) блок; мобайл: аккордеон под карточками.
Контент:
«Персональный Telegram-бот: +0,5 цента/запрос»
«Удаление логотипа с виджета: +0,5 цента/запрос»
В monthly-режиме блок свёрнут и показывает: «Эти опции включены в месячную подписку».
6) Кнопки «Подписаться сейчас»
В каждой карточке — CTA-кнопка, визуально согласованная с дизайном.
Поддержать два способа задания ссылок:
Через конфиг в script.js.
Через data-payment-url-usage и data-payment-url-monthly на карточке.
При клике подставлять ссылку согласно текущему режиму биллинга.
7) Подсказки по ховеру (tooltips)
Наведение на элементы списка «что входит» открывает подсказку.
Источник контента: блоки <template id="tooltip-KEY">...</template> в index.html или data-tooltip="KEY" на элементе + словарь в script.js (любой один основной способ, второй — резервный).
Поведение:
Плавное появление/скрытие, задержка 150–200 мс.
Умное позиционирование (автосмена стороны при нехватке места).
На мобайле показ по тапу, закрытие вторым тапом вне.
8) Типографика и стили
Сохранить иерархию, сетку и цветовую схему референса.
Шрифты Inter/Montserrat через Google Fonts (или локально).
Видимые ховеры/фокусы для CTA и элементов с подсказками.
9) Конфигурация в script.js
const CONFIG = {
  billingDefault: 'usage', // 'usage' | 'monthly'
  paymentLinks: {
    pro: {
      usage:   'https://pay.example.com/pro-usage',    // placeholder
      monthly: 'https://pay.example.com/pro-monthly'
    },
    premium: {
      usage:   'https://pay.example.com/premium-usage',
      monthly: 'https://pay.example.com/premium-monthly'
    }
  },
  tooltips: {
    // 'personal-manager': 'Текст подсказки ...',
  }
};
Если у карточки есть data-payment-url-usage/data-payment-url-monthly, они имеют приоритет над CONFIG.paymentLinks.
10) Логика JS (кратко)
Инициализация: прочитать #billing= → применить; иначе взять из localStorage или CONFIG.billingDefault.
Обработчик тумблера: переключать класс mode-usage/mode-monthly на <body> и обновлять тексты.
Тексты цен менять через элементы с классами: .price-per-request, .price-monthly, .old-price-monthly, и т.д.
Тултипы: делегирование событий от общего контейнера, позиционирование через getBoundingClientRect.
11) Доступность (a11y)
Тумблер — role="switch", состояние aria-checked.
Подсказки — role="tooltip", привязка через aria-describedby, закрытие по Esc.
12) Адаптив
≥1200px: карточки в ряд, usage-блок справа (sticky).
768–1199px: карточки столбцом, usage-блок под ними.
≤767px: один столбец, подсказки — как popover-плашки.
13) Приёмка (чек-лист)
 Тумблер меняет режимы, состояние сохраняется и восстанавливается.
 В usage видно: «Персональный Telegram-бот» и «Удаление логотипа» = +0,5 цента/запрос, боковой блок присутствует.
 В monthly те же опции помечены «Входит», боковой блок свёрнут с ремаркой «включено».
 PRO: старая «8 центов» (зачёркнуто) → текущая «7 центов за запрос»; при monthly — «459 долларов» → «399 долларов в месяц».
 PREMIUM: «30 центов за запрос»; при monthly — «1499 долларов в месяц».
 Кнопки «Подписаться сейчас» берут верные ссылки для текущего режима.
 Подсказки открываются по ховеру/тачу, контент — из index.html.
 Мобайл-вид корректен; нет горизонтального скролла.
